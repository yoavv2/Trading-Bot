<objective>
Close the one remaining Phase 11 verification gap by making broker-fill deduplication selective to the
current broker sync batch, proving that the existing named unique index is used, and only then marking
PERF-03 Complete.

`_ingest_paper_fills` currently executes `select(PaperFill.broker_fill_id)` with no predicate on every
`sync_paper_state` call. At realistic history size PostgreSQL correctly chooses a full Seq Scan because
the statement asks for every historical ID; adding another index cannot improve that shape. The existing
`uq_paper_fills_broker_fill_id` unique index is sufficient once the statement filters to the IDs in the
current `broker_fills` argument.

Purpose: Make dedup lookup cost depend on the current broker response (normally capped by Alpaca's
`list_fills(page_size=500)` default), not accumulated history, while preserving idempotent ingestion and
remaining safe if a caller supplies a much larger batch.

Output: A batch-scoped and chunked lookup in `paper_execution.py`, regression tests with real historical
rows plus empty/duplicate/oversized input, the existing Seq Scan documentation test repurposed into a
named-index EXPLAIN assertion, and PERF-03 marked Complete only after proof is green.
</objective>
