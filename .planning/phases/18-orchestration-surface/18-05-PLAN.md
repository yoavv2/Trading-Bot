<threat_model>
## OWASP ASVS L1 / Trust Boundaries

| Boundary | Description |
|---|---|
| Local shell → worker parser/dispatch | A legacy command can bypass HTTP idempotency and directly initiate financial mutations. |
| API/CLI adapters → application/domain layers | Adapter-local semantics can fork validation, lifecycle, or transaction behavior. |
| Application orchestration → domain services | Infrastructure may orchestrate Jobs, but domain services must never import upward. |

## STRIDE Threat Register

| Threat ID | Category | Severity | Component | Disposition | Mitigation Plan |
|---|---|---:|---|---|---|
| T-18-20 | Elevation of Privilege | Critical | worker parser/DISPATCH | mitigate | Exact retained whitelist and removed blacklist asserted at runtime/AST; argparse rejection for every bypass command. |
| T-18-21 | Tampering | High | route/CLI adapter logic | mitigate | AST checks forbid persistence/lifecycle/domain-operation logic; retained `run-jobs` thinness pinned. |
| T-18-22 | Repudiation | High | parallel CLI execution path | mitigate | Remove registrations so all future mutations receive HTTP idempotency/audit linkage. |
| T-18-23 | Elevation of Privilege | High | services import boundary | mitigate | Keep existing auto-scoped blanket unchanged; allowed orchestration lives outside `services` and has its own inverse boundary. |
| T-18-24 | Denial of Service | Medium | regression-test removal | mitigate | Rewrite lock test at service boundary rather than deleting the safety invariant. |
| T-18-SC | Tampering | Low | package supply chain | accept | No install task. |
