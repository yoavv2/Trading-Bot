<threat_model>
## OWASP ASVS L1 / Trust Boundaries

| Boundary | Description |
|---|---|
| Untrusted mutation identity → orchestration service | Headers/bodies determine whether financial work is created, replayed, or rejected. |
| Orchestration service → Job framework/PostgreSQL | One transaction composes registry validation, Job mutation, idempotency, and audit facts. |
| Job reference → future client | Links must not trust Host/proxy input or embed unbounded evolving data. |

## STRIDE Threat Register

| Threat ID | Category | Severity | Component | Disposition | Mitigation Plan |
|---|---|---:|---|---|---|
| T-18-09 | Tampering | Critical | concurrent `submit` | mitigate | Named unique constraint + savepoint rollback of losing candidate + independent-session race tests. |
| T-18-10 | Tampering | High | canonical fingerprints | mitigate | Sorted compact JSON, `allow_nan=False`, SHA-256, endpoint-specific operation material, mismatch conflict. |
| T-18-11 | Repudiation | High | cancellation repeats | mitigate | Route eligible requests through `request_cancellation`; pending/CANCELLED repeat tests pin first audit facts. |
| T-18-12 | Spoofing | High | reference links | mitigate | Build fixed relative paths only; no Request/Host/proxy dependency exists in this layer. |
| T-18-13 | Denial of Service | Medium | key/reason/payload inputs | mitigate | 255/500 bounds and JSON canonicalization failure before writes; HTTP body-size policy remains framework/deployment-owned. |
| T-18-14 | Elevation of Privilege | High | architecture boundary | mitigate | Place application orchestration outside `services`; import enforcement in Plan 05 blocks HTTP/worker/domain-service imports and keeps JOB-04 blanket unchanged. |
| T-18-SC | Tampering | Low | package supply chain | accept | Uses stdlib hashlib/json and existing SQLAlchemy only; no installation. |
