<objective>
Turn the paper-submission path into a deterministic idempotent intent pipeline by deriving `client_order_id` from material order identity, reusing persisted intents on retry, and making new-version behavior explicit once the broker has touched an order.

Purpose: This plan closes the duplicate-submission hole. The same trade intent must map to the same broker identity across reruns and restarts, while materially new attempts must still stay traceable as deliberate new versions.
Output: Deterministic order identity helpers, idempotent submission/retry behavior, client-order-first broker matching, and operator-visible reuse/version notes.
</objective>
