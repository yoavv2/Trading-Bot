<objective>
Establish the typed contracts that the entire reconciliation rewrite is built against: a
closed `ReconciliationFinding` enum, typed local snapshot dataclasses that mirror the
already-typed broker snapshots, and a hashable `ReconciliationIdentity` key over
`(symbol, account, side)`. This is the interface-first foundation — the matcher (09-02)
and orchestrator (09-03) implement against these types, so they land first with no
implementation logic beyond construction/derivation.

Purpose: Eliminate string-classified findings (RECON-07) and the raw-ORM / dict-of-strings
snapshot boundary (RECON-05) at the type level, so downstream code physically cannot
reintroduce them.
Output: `reconciliation_types.py` with the closed enum, four typed local snapshots, the
identity key, and pure derivation helpers — fully unit-tested, no DB or broker access.
</objective>
