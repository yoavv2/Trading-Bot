Add `cascade_dependency_outcome(session, *, terminal_job_id: uuid.UUID) -> list[uuid.UUID]` to `dependencies.py`.

Precondition: the named Job has just reached FAILED or CANCELLED. If its status is SUCCEEDED, RUNNING, or QUEUED, return an empty list without writing â€” this makes an accidental call harmless.

Behavior (D-04): perform a breadth-first traversal outward over the reverse dependency edges (every Job that depends on the terminal Job, then every Job that depends on those, and so on). For each descendant still in `QUEUED` â€” and only QUEUED, since D-04 says "still-unstarted"; a descendant already RUNNING is left alone because a running Job is cancelled cooperatively through plan 17-06's path, not silently terminated here â€” call `apply_job_transition` with `event_type=JobEventType.CANCELLED` and a `JobTransitionRequest` carrying:
  `cancellation_cause = JobCancellationCause.DEPENDENCY_FAILED` when the immediate blocking Job's status is FAILED, or `DEPENDENCY_CANCELLED` when it is CANCELLED (D-04);
  `blocking_job_id` = the immediate parent that blocked this descendant, and `blocking_job_status` = that parent's terminal status (D-05);
  `root_cause_job_id` = `terminal_job_id`, the root failed-or-cancelled ancestor of the whole cascade (D-05);
  `failure_reason = None` â€” a dependency cascade is a cancellation, never an infrastructure failure (D-01);
  `requested_by = None` and `reason` set to a generated string naming the blocking Job and its terminal status, so the causal chain is human-readable as well as queryable;
  `details` including the blocking Job ID, the root cause Job ID, and the traversal depth.

Idempotency is load-bearing and must mirror `reclaim_stale_runs`: once a descendant leaves QUEUED it no longer matches the traversal filter, so a second call over the same subgraph transitions nothing and returns an empty list. Do not add a separate "already cascaded" flag â€” the status filter is the mechanism.

Cycle safety: even though cycles are unrepresentable, the traversal must carry a visited set and skip already-visited Job IDs, so a corrupted graph produces a bounded traversal rather than an infinite loop.

Return the list of cancelled descendant Job IDs in traversal order. The caller (plan 17-07's runner, on every terminal FAILED/CANCELLED transition) owns the transaction; this function flushes and never commits.

Update the module docstring to state that a dependent Job must never be left stranded in QUEUED behind a dead dependency (D-04) and that this function is the only mechanism preventing that.
