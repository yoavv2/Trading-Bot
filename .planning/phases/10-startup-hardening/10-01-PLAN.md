<objective>
Build the pure, entrypoint-agnostic config validation core: a closed `ExecutionMode` enum, a typed `ConfigValidationError` carrying the failed field name and expected shape, and a `validate_config(settings, *, mode)` function that runs all CFG-01/02/03/05/07 checks and raises a single aggregated, actionable error. This plan produces NO wiring into entrypoints and NO process-exit behavior — that is plan 10-05. This plan is the contract 10-05 consumes.

Purpose: Startup must refuse invalid config with a message naming the failed field, but the *decision of what is valid* must be a pure, unit-testable function decoupled from where the process boots.
Output: `core/config_validation.py` + full unit test suite.
