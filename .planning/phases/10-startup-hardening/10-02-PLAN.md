<objective>
Build the log-sanitization core: a `sanitize()` function that redacts credentials, API keys, tokens, connection URLs containing passwords, and Authorization headers, and masks broker order IDs to last-6 by default (full ID only behind an explicit debug flag) — and route the existing `emit_structured_log` helper's payload through it so there is a single sanitization chokepoint. Also expose a `get_logger()` wrapper as the API contract that plan 10-06 will migrate all execution/reconciliation/config paths onto.

This plan does NOT migrate any caller off `logging.getLogger` and does NOT add the enforcement/import-boundary tests — that is plan 10-06. This plan ships the contract 10-06 consumes.

Purpose: LOG-06 requires that no emitted log line under default config leaks secrets or full broker IDs. That is only enforceable if there is exactly one place every payload is sanitized.
Output: `core/log_sanitizer.py`, sanitizer wired into `core/logging.py`, unit tests.
</objective>

  <name>Task 1: sanitize() redaction rules for credentials, tokens, conn-URLs, and auth headers</name>
  <files>src/trading_platform/core/log_sanitizer.py, tests/test_log_sanitizer.py</files>
  <behavior>
    - `sanitize({"password": "hunter2"})` → value replaced with a redaction marker (e.g. `"[REDACTED]"`); the key `password` no longer maps to the secret (LOG-03).
    - Keys matching credential/token patterns (`password`, `api_key`, `api_secret`, `secret`, `token`, `authorization`) are redacted case-insensitively, at any nesting depth in the dict (LOG-03/LOG-04).
    - A connection URL string containing an embedded password (`postgresql+psycopg://user:hunter2@host/db`) is redacted so the password substring does not survive (LOG-03).
    - An `Authorization` header value (e.g. `{"headers": {"Authorization": "Bearer abc"}}`) is redacted (LOG-04).
    - A plain string value that itself embeds `password=...` or `api_key=...` is scrubbed of the secret token (defends against a secret placed in a free-text message field).
    - sanitize is pure and returns a NEW structure; the input dict is not mutated.
  </behavior>
  <action>
    Create `core/log_sanitizer.py`. Implement `sanitize(payload: Any) -> Any` that recursively walks dicts/lists: redact values whose key matches a compiled case-insensitive `SENSITIVE_KEY_PATTERN` (password|passwd|api[_-]?key|api[_-]?secret|secret|token|authorization|auth); for string values, regex-scrub embedded `key=value` secret pairs and password-bearing connection URLs (`://user:PASSWORD@`). Use a single `REDACTION = "[REDACTED]"` constant. Keep it dependency-free (stdlib `re` only). Write unit tests covering each rule, nested dicts, and no-mutation-of-input.
  </action>
