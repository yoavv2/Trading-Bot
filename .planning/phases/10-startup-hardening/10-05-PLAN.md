<objective>
Wire the pure config validator (plan 10-01) into every process entrypoint so the process refuses to boot on invalid config: a startup gate that builds the raw settings payload, runs validate_config (which returns a validated Settings) plus a DB-reachability preflight and, on any failure, prints the single actionable field-naming message to stderr and exits with a non-zero code — invoked BEFORE any domain service initializes at the FastAPI lifespan, every worker subcommand, and the bootstrap flow.

Because validate_config owns Settings construction from the raw payload (10-01), the gate must run it BEFORE the entrypoint's own load_settings() call, so an out-of-range tolerance surfaces as a typed field-named exit rather than a raw pydantic traceback from load_settings.

Purpose: CFG-04 (unreachable DB = exit before domain code) and CFG-06 (all validation before service init; one failure prevents init) are the enforcement half of config hardening — 10-01 decides validity, this plan makes the process act on it.
Output: core/startup.py gate + entrypoint wiring + tests.
</objective>
