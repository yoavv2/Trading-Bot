<objective>
Wire the pure config validator (plan 10-01) into every process entrypoint so the process refuses to boot on invalid config: a startup gate that runs `validate_config` plus a DB-reachability preflight and, on any failure, prints the single actionable field-naming message to stderr and exits with a non-zero code — invoked BEFORE any domain service initializes at the FastAPI lifespan, every worker subcommand, and the bootstrap flow.

Purpose: CFG-04 (unreachable DB = exit before domain code) and CFG-06 (all validation before service init; one failure prevents init) are the enforcement half of config hardening — 10-01 decides validity, this plan makes the process act on it.
Output: `core/startup.py` gate + entrypoint wiring + tests.
</objective>
